## Installing Your First Tool

Let's walk through installing **Nmap**, a popular network scanning tool:

<Steps>
  <Step title="Select Information Gathering">
    From the main menu, type `1` and press Enter to access Information Gathering tools:
    
    ```
    Choose a tool to proceed: 1
    ```
  </Step>

  <Step title="Browse Available Tools">
    You'll see a submenu with all information gathering tools:
    
    ```
    ╭─────────────── Information gathering tools ───────────────╮
    │                                                           │
    │   0  Network Map (nmap)                                   │
    │   1  Dracnmap                                             │
    │   2  Port scanning                                        │
    │   3  Host to IP                                           │
    │   4  Xerosploit                                           │
    │   5  RED HAWK (All In One Scanning)                       │
    │   6  ReconSpider (For All Scanning)                       │
    │   7  IsItDown (Check Website Down/Up)                     │
    │   8  Infoga - Email OSINT                                 │
    │   9  ReconDog                                             │
    │  10  Striker                                              │
    │  11  SecretFinder (like API & etc)                        │
    │  12  Find Info Using Shodan                               │
    │  13  Port Scanner - rang3r (Python 2.7)                   │
    │  14  Port Scanner - Ranger Reloaded (Python 3+)           │
    │  15  Breacher                                             │
    │  99  Back to Main Menu                                    │
    │                                                           │
    ╰───────────────────────────────────────────────────────────╯
    ```
  </Step>

  <Step title="Select Nmap">
    Type `0` to select Nmap:
    
    ```
    Choose a tool to proceed: 0
    ```
    
    HackingTool will:
    1. Check if Nmap is already installed
    2. Download and install Nmap if needed
    3. Display installation status
    
    <Tip>
    Most tools are cloned from GitHub repositories and installed automatically with their dependencies.
    </Tip>
  </Step>

  <Step title="Verify Installation">
    After installation, you can verify Nmap is working:
    
    ```bash
    nmap --version
    ```
    
    Expected output:
    ```
    Nmap version 7.94 ( https://nmap.org )
    Platform: x86_64-pc-linux-gnu
    Compiled with: liblua-5.3.6 openssl-3.0.11
    ```
  </Step>
</Steps>

## Running Your First Scan

Now that Nmap is installed, let's perform a basic network scan:

<Warning>
**Legal Notice**: Only scan networks and systems you own or have explicit permission to test. Unauthorized scanning is illegal.
</Warning>

<Steps>
  <Step title="Scan Your Local Network">
    Identify devices on your local network:
    
    ```bash
    sudo nmap -sn 192.168.1.0/24
    ```
    
    **Example Output:**
    ```
    Starting Nmap 7.94 ( https://nmap.org )
    Nmap scan report for 192.168.1.1
    Host is up (0.0012s latency).
    MAC Address: AA:BB:CC:DD:EE:FF (Router Manufacturer)
    
    Nmap scan report for 192.168.1.100
    Host is up (0.023s latency).
    MAC Address: 11:22:33:44:55:66 (Device Manufacturer)
    
    Nmap done: 256 IP addresses (12 hosts up) scanned in 3.45 seconds
    ```
  </Step>

  <Step title="Perform a Port Scan">
    Scan common ports on a specific host:
    
    ```bash
    sudo nmap -sV 192.168.1.100
    ```
    
    **Example Output:**
    ```
    Starting Nmap 7.94 ( https://nmap.org )
    Nmap scan report for 192.168.1.100
    Host is up (0.00055s latency).
    Not shown: 997 closed ports
    PORT    STATE SERVICE VERSION
    22/tcp  open  ssh     OpenSSH 8.9p1 Ubuntu 3ubuntu0.1
    80/tcp  open  http    Apache httpd 2.4.52
    443/tcp open  ssl/http Apache httpd 2.4.52
    
    Service detection performed. Please report any incorrect results.
    Nmap done: 1 IP address (1 host up) scanned in 6.78 seconds
    ```
  </Step>

  <Step title="Save Scan Results">
    Export results to a file for documentation:
    
    ```bash
    sudo nmap -sV -oN scan_results.txt 192.168.1.100
    ```
    
    This creates `scan_results.txt` with the full scan output.
  </Step>
</Steps>

## Common Workflows

### Workflow 1: Reconnaissance Phase

<Steps>
  <Step title="Enable Anonymity">
    **Category 0** → Anonsurf
    ```bash
    anonsurf start
    ```
  </Step>

  <Step title="Gather Target Information">
    **Category 1** → Nmap, ReconDog, or RED HAWK for initial recon
  </Step>

  <Step title="Enumerate Subdomains">
    **Category 6** → Sublist3r for subdomain discovery
  </Step>

  <Step title="Search for Vulnerabilities">
    **Category 4** → SQLMap for SQL injection testing
    **Category 14** → XSStrike for XSS vulnerabilities
  </Step>
</Steps>

### Workflow 2: Password Testing

<Steps>
  <Step title="Generate Wordlist">
    **Category 2** → Cupp (Common User Passwords Profiler)
    ```bash
    cupp -i  # Interactive mode
    ```
  </Step>

  <Step title="Use Wordlist">
    Apply the generated wordlist to brute force tools in Category 16
  </Step>
</Steps>

### Workflow 3: Wireless Security Testing

<Steps>
  <Step title="Monitor Mode">
    Enable monitor mode on your wireless adapter
  </Step>

  <Step title="Launch Wifite">
    **Category 3** → Wifite for automated WiFi attacks
    ```bash
    sudo wifite
    ```
  </Step>

  <Step title="Capture Handshakes">
    Wifite automatically captures WPA handshakes
  </Step>

  <Step title="Crack Passwords">

</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Tool Installation Fails">
    **Problem**: Git clone or installation errors
    
    **Solutions**:
    1. Check internet connectivity: `ping github.com`
    2. Verify you have git installed: `git --version`
    3. Ensure sufficient disk space: `df -h`
    4. Try manually cloning: `cd /home/hackingtool && git clone [tool-url]`
  </Accordion>

  <Accordion title="Tool Won't Run">
    **Problem**: Command not found after installation
    
    **Solutions**:
    1. Navigate to tool directory: `cd /home/hackingtool/[tool-name]`
    2. Run directly: `python3 tool.py` or `./tool.sh`
    3. Check dependencies: `pip3 install -r requirements.txt`
    4. Verify permissions: `chmod +x tool.sh`
  </Accordion>

  <Accordion title="Permission Denied Errors">
    **Problem**: Cannot write to directories or execute tools
    
    **Solution**: Always run HackingTool with sudo:
    ```bash
    sudo hackingtool
    ```
  </Accordion>

  <Accordion title="Menu Not Displaying Correctly">
    **Problem**: Broken characters or formatting issues
    
    **Solutions**:
    1. Ensure terminal supports UTF-8: `echo $LANG`
    2. Install missing fonts for emoji support
    3. Use a modern terminal emulator (Terminator, Tilix, Kitty)
  </Accordion>
</AccordionGroup>
